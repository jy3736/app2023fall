AI2 UI Component Selection (Palette)

User Interface Palette

The User Interface Palette is where you will find all the components you need to build the user interface (UI) of your app. The components are grouped by functionality, making it easy to find the right component for your needs.

Component Selection

To select a component, simply click on it in the palette. The component will then be added to the Viewer, which is the area where you can design your app's UI.

Interface Layout

Once you have added some components to the Viewer, you can start to arrange them in a layout. Layouts help to format components so they are attractive and organized on the screen.

There are many different types of layouts available in App Inventor. Some of the most common layouts include:

  • HorizontalArrangement: Arranges components in a horizontal line.
  • VerticalArrangement: Arranges components in a vertical line.
  • RelativeLayout: Allows you to position components relative to each other.
  • TableArrangement: Arranges components in a table format.

To add a layout to your app, simply click on the layout you want to use in the palette and drag it to the Viewer. The layout will then be added to the Viewer and you can start to add components to it.

Component Properties

Once you have added a component to a layout, you can change its properties to customize its appearance and behavior. To change a component's properties, simply select the component in the Viewer and its properties will be displayed in the Properties panel on the right side of the screen.

Some of the most common component properties include:

  • Text: The text that is displayed on the component.
  • Background color: The background color of the component.
  • Font size: The size of the font used to display text on the component.
  • Enabled: Whether or not the component is enabled.

In the absence of assistance with interface layout elements, it is not possible to align multiple elements side by side in the same row.

Use the Horizontal Arrangement layout element to arrange multiple components horizontally.

Use the Horizontal Arrangement layout element to arrange multiple components horizontally.

Instructions:

  1. Open App Inventor 2 and create a new project.
  1. Drag a Horizontal Arrangement component from the Palette onto the Designer.
  1. Add one or more components to the Horizontal Arrangement, such as Labels, Buttons, Images, or TextBoxes.
  1. Click the Preview button to see how the components are arranged.
  1. Use the Properties panel to change the appearance of the Horizontal Arrangement, such as its alignment, height, and width.

Details:

  • The Horizontal Arrangement component is a formatting element that allows you to arrange other components horizontally.
  • Components in a Horizontal Arrangement are arranged from left to right, with the first component appearing on the left and the last component appearing on the right.
  • The components in a Horizontal Arrangement are vertically centered.
  • You can change the alignment of the components in a Horizontal Arrangement by setting the Alignment property to Left, Center, or Right.
  • You can also change the height and width of the Horizontal Arrangement by setting the Height and Width properties, respectively.

Utilize the Vertical Arrangement layout component to arrange multiple elements vertically.

Utilize the Vertical Arrangement layout component to arrange multiple elements vertically.

Instructions:

  1. Create a new App Inventor 2 project.
  1. Add a Vertical Arrangement component to the screen.
  1. Add multiple elements to the Vertical Arrangement component, such as labels, buttons, images, and text inputs.
  1. Arrange the elements in the order that you want them to appear on the screen.
  1. Set the properties of the Vertical Arrangement component, such as the alignment and padding.
  1. Preview the app and make sure that the elements are arranged correctly.

Additional Details:

  • The Vertical Arrangement component is a formatting element that arranges its child components vertically, from top to bottom.
  • The width of the Vertical Arrangement component is determined by the widest child component whose width property is not set to Fill Parent.
  • The height of the Vertical Arrangement component is determined by the total height of its child components.
  • You can use the alignment property of the Vertical Arrangement component to specify how the child components are aligned horizontally. The choices are left aligned, horizontally centered, and right aligned.
  • You can use the padding property of the Vertical Arrangement component to add space around the child components.

Adjust the layout using the 'Interface Layout' and 'Component' Width/Height properties.

Adjust the layout using the 'Interface Layout' and 'Component' Width/Height properties

App Inventor 2 provides two ways to adjust the layout of your app:

  • Interface Layout: This is the overall layout of your screen, and it is controlled by the Screen and Arrangement components.
  • Component Width/Height: This controls the size of individual components on your screen.

To adjust the layout using the Interface Layout, you can use the following properties:

  • Screen:
    • Orientation: This determines whether the screen is displayed in landscape or portrait mode.
    • Title: This is the text that is displayed in the title bar of the app.
    • Background Color: This is the color of the background of the screen.
  • Arrangement:
    • AlignHorizontal: This determines how the components in the arrangement are aligned horizontally.
    • AlignVertical: This determines how the components in the arrangement are aligned vertically.
    • BackgroundColor: This is the color of the background of the arrangement.

To adjust the layout using the Component Width/Height properties, you can use the following properties:

  • Width: This determines the horizontal width of the component, in pixels.
  • Height: This determines the vertical height of the component, in pixels.
  • WidthPercent: This determines the horizontal width of the component as a percentage of the screen width.
  • HeightPercent: This determines the vertical height of the component as a percentage of the screen height.

App Inventor 2 Exercise: Designing and Previewing Your App

One of the benefits of using App Inventor 2 is that you can design and preview your app without having to write any code. This makes it easy to experiment with different ideas and to get a feel for how your app will look and work before you start coding.

To design your app, you will use the App Inventor 2 designer. The designer is a web-based tool that allows you to create the user interface for your app by dragging and dropping components onto a canvas. You can also use the designer to set the properties of each component, such as its size, color, and text.

Once you have designed your app, you can preview it on your phone by uploading it to the App Inventor 2 cloud server. To do this, you will need to create an App Inventor 2 account and connect your phone to the cloud server.

Once your app is uploaded, you can open it on your phone by using the App Inventor 2 companion app. The companion app is available for free on the Google Play Store.

When you preview your app on your phone, you will be able to see how the user interface looks and works. You will also be able to test any functionality that you have implemented.

It is important to note that the App Inventor 2 designer only presents the layout of the interface. The actual presentation on the phone may vary depending on the device's screen size and other factors. To get a more accurate preview of your app, it is recommended that you test it on a variety of devices.

Here are some additional tips for designing and previewing your App Inventor 2 app:

  • Use the App Inventor 2 documentation to learn more about the designer and the different components that you can use.
  • Experiment with different ideas and layouts to see what works best for your app.
  • Use the preview feature to test your app on different devices.
  • Get feedback from others on how your app looks and works.

Once you are satisfied with the design of your app, you can start coding to implement the functionality.

MIT App Inventor built-in blocks are the basic building blocks of App Inventor programs. They are available regardless of which components are in your project. Built-in blocks can be used to perform a wide variety of tasks, including:

  • Controlling the flow of a program
  • Performing mathematical operations
  • Manipulating text and lists
  • Communicating with the device's hardware and sensors

Built-in blocks are organized into the following categories:

  • Control blocks: These blocks allow you to control the flow of a program, such as by creating loops, conditional statements, and procedures.
  • Logic blocks: These blocks allow you to perform logical operations, such as AND, OR, and NOT.
  • Math blocks: These blocks allow you to perform mathematical operations, such as addition, subtraction, multiplication, and division.
  • Text blocks: These blocks allow you to manipulate text, such as by converting it to upper or lowercase, extracting a substring, or joining two strings together.
  • Lists blocks: These blocks allow you to manipulate lists, such as creating a new list, adding or removing items from a list, and sorting a list.

To use a built-in block, simply drag it from the Blocks Palette onto the Designer. You can then connect the block to other blocks to create a program.

Here are some examples of how built-in blocks can be used:

  • To create a button that opens a new screen when it is clicked, you would use a Button.Click event block and a Screen.Open block.
  • To calculate the area of a triangle, you would use a Math.Multiply block to multiply the base and height of the triangle.
  • To display a message to the user, you would use a Label.Text block to set the text of a label to the message.
  • To create a list of all the files in a directory, you would use a File.List block.

Built-in blocks are a powerful tool that can be used to create a wide variety of apps. By learning how to use built-in blocks, you can create apps that can perform all sorts of tasks, from simple to complex.

Exercise: Open an existing App Inventor project or create a new one.

In this exercise, we will explore how to work with MIT App Inventor, a powerful platform for creating mobile applications without the need for extensive coding. In this session, we will focus on two essential tasks: opening an existing App Inventor project and creating a new one.

Opening an Existing Project

  1. Accessing App Inventor:
    • Launch a web browser and navigate to the MIT App Inventor website (https://appinventor.mit.edu/).
    • Log in to your App Inventor account using your credentials.
  1. Opening an Existing Project:
    • After logging in, you will be directed to the "My Projects" page.
    • Locate the project you want to open in the project list.
    • Click on the project's name to open it in the App Inventor interface.
    • You are now ready to work on your existing project.

Creating a New Project

  1. Starting a New Project:
    • From the "My Projects" page, click on the "Start New Project" button.
    • A dialog box will appear. Enter a suitable name for your new project and click "OK."
  1. The App Inventor Interface:
    • Once your new project is created, you will be taken to the App Inventor interface.
    • Here, you can see the components, blocks editor, and the viewer for your app's design.
    • You are now ready to start building your new app from scratch.

In today's session, we've learned how to open an existing App Inventor project and create a new one. Whether you're continuing your work on a previous project or embarking on a new mobile app journey, App Inventor provides a user-friendly platform for all your app development needs.

Exercise: User Interface (UI) Design for AI2 Projects

In this exercise, we will explore the fundamentals of User Interface (UI) Design for AI2 (MIT App Inventor 2) projects. Just as "Hello, world!" is the simplest program in many programming languages, we'll start with a basic UI design to get you started.

Creating a Simple UI

  1. Accessing App Inventor:
    • Open a web browser and go to MIT App Inventor (https://appinventor.mit.edu/).
    • Log in to your App Inventor account if you haven't already.
  1. Starting a New Project:
    • Click on "Start New Project."
    • Enter a project name, and make sure it's set to "Phone/Tablet" for the target device.
    • Click "OK" to create your project.
  1. Designing the UI:
    • In the App Inventor interface, you'll see a visual designer. It's divided into two main sections: the Viewer and the Palette.
    • Drag and drop components from the Palette onto the Viewer to create your UI.
    • For our "Hello, world!" example, add:
      • Label: Drag a Label component to the Viewer.
      • Button: Drag a Button component to the Viewer below the label.
  1. Customize Components:
    • Select the Label component, and in the Properties panel, change the "Text" property to "Hello, world!"
    • For the Button component, set the "Text" property to "Click Me!"
  1. Testing Your UI:
    • Click on the "Blocks" button to open the Blocks editor.
    • In the Blocks editor, create a "when Button.Click" block.
    • Inside this block, use the "Set Label.Text" block to change the text of the label to "Hello, App Inventor!" when the button is clicked.
    • Click the "Connect to device" button to test your app on an Android device or emulator.
  1. Running Your App:
    • Install and run the app on your device or emulator.
    • When you click the "Click Me!" button, the label should change to "Hello, App Inventor!"
  1. Saving Your Project:
    • Remember to save your project by clicking the floppy disk icon in the top left corner.

In this introductory session, we've created a simple UI for an AI2 project, just like the "Hello, world!" example in various programming languages. This basic example showcases the essential components and steps required for UI design in App Inventor.

As you progress, you can enhance your UI and add more functionality to your AI2 projects. Experiment with different components and explore the vast possibilities of mobile app development using MIT App Inventor.

Exercise: Blockly Programming for AI2 UI Components

In this exercise, we will explore how to create a simple click counter using Blockly programming in AI2 (MIT App Inventor 2). This hands-on session will help you understand how to manipulate variables and UI components to build interactive mobile apps.

Setting Up the Environment:

  1. Open a web browser and navigate to MIT App Inventor (https://appinventor.mit.edu/).
  1. Log in to your App Inventor account if you haven't already.

Creating a New Project:

  1. Click on "Start New Project."
  1. Enter a project name and ensure it's set to "Phone/Tablet" for the target device.
  1. Click "OK" to create your project.

Designing the UI:

  1. In the App Inventor interface, locate the visual designer, which comprises the Viewer and the Palette.
  1. Add the following UI components to the Viewer:
    • Label: Drag and drop a Label component to display the click count.
    • Button: Add a Button component with the text "Click Me!"
  1. Customize Component Properties:
    • Select the Label and set its "Text" property to "Clicks: 0"

Blockly Programming:

  1. Click the "Blocks" button to open the Blocks editor.
  1. Create a Blockly variable called "ClickCount."
  1. Set the initial value of "ClickCount" to 0.
  1. Create a Blockly block that responds to the Button's "Click" event.
  1. Inside this block, use Blockly logic to:
    • Increment the "ClickCount" variable by 1.
    • Update the text of the Label to display the new click count.

Testing Your App:

  1. Connect to a device or emulator using the "Connect to device" button.
  1. Install and run your app on the device or emulator.
  1. Test the app by clicking the "Click Me!" button. Observe how the click count increases with each click.

Conclusion:

In this exercise, we've created a simple click counter using Blockly programming in AI2. By manipulating variables and UI components, you've learned how to build interactive mobile apps. This exercise serves as a foundation for more advanced app development tasks in MIT App Inventor.

Feel free to experiment further with Blockly and UI components to expand your skills in mobile app development.
